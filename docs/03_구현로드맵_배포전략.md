# 홈페이지 v2 구현 로드맵 및 배포 전략

## 1) 목표
- 1차: 정적 우선으로 빠르게 공개 가능한 완성형 홈페이지 출시
- 2차: 무료 동적 기능(조회수, 검색 고도화, 문의 폼, 경력 통계) 단계적 추가

## 2) 권장 기술 선택
- 프레임워크: Astro (정적 우선 + 다국어 + 콘텐츠 확장성)
- 스타일: CSS 변수 기반 토큰 + 유틸리티 클래스 병행
- 데이터 소스: CSV -> 빌드 시 JSON 변환
- 배포: GitHub Pages (1차), 필요 시 Cloudflare 연동 (2차)

## 3) 개발 단계

### Phase 0. 설계 고정
- IA/와이어프레임 승인
- 컴포넌트 목록 확정
- 다국어 키 구조 확정

### Phase 1. 기본 골격
- 라우팅: `/ko`, `/ja`, `/en`
- 공통 레이아웃, 헤더, 푸터
- 기본 테마 토큰 (`#87CEEB` 중심)

### Phase 2. 데이터 연결
- `personal_info.csv` 파싱 -> Home/Profile 매핑
- `career_data.csv` 파싱 -> Career 타임라인/필터
### Phase 3. 블로그
- 카테고리형 목록 페이지
- 상세 페이지 템플릿
- 태그/검색(정적 인덱스 기반 1차)

### Phase 4. QA 및 배포
- 모바일/데스크톱 반응형 점검
- 다국어 링크/폴백 점검
- GitHub Pages 배포 파이프라인 구성

## 4) 무료 동적 기능 확장안 (2차)

### A안: GitHub Pages + Cloudflare Workers
- 장점:
  - 프론트 정적 배포 유지
  - 필요한 API만 서버리스로 분리 가능
  - 무료 구간에서 시작하기 좋음
- 대상 기능:
  - 조회수 카운트 API
  - 문의 폼 수집 API
  - 고급 검색 인덱스 API

### B안: Cloudflare Pages + Functions 일원화
- 장점:
  - 프론트/백엔드 한 플랫폼에서 관리
  - 캐시/엣지 성능 강점
- 대상 기능:
  - A안과 동일 + 경력 통계 API

## 5) 기능 우선순위 반영 (요청사항 기준)
- 1순위: 조회수/방문 통계
- 2순위: 블로그 검색/태그 고도화
- 3순위: 문의 폼
- 4순위: 경력 필터 고도화(복합 필터 + 집계)

## 6) GitHub Pages 배포 체크리스트
- 정적 빌드 산출물 디렉터리 고정
- 404 처리(다국어 경로 대응)
- 커스텀 도메인 사용 시 CNAME 설정
- 배포 브랜치/액션 워크플로 점검

## 7) 성능/품질 지표
- LCP 2.5s 이하(모바일)
- 언어 전환 후 누락 키 0건
- Career 필터 반응 시간 100ms 내 체감
- 접근성 핵심 항목(키보드/명도 대비) 충족

## 8) 리스크와 대응
- 리스크: CSV 원문 데이터의 형식 불일치 가능
  - 대응: 변환 단계에서 스키마 검증 및 기본값 폴백
- 리스크: 3개 국어 번역 누락
  - 대응: 키 검증 스크립트 + `ko` 폴백
- 리스크: 동적 기능 무료 한도 초과
  - 대응: API 캐시/레이트 리밋/집계 주기 조정
