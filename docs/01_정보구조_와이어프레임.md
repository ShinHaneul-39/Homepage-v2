# 홈페이지 v2 정보구조(IA) 및 와이어프레임

## 1) 프로젝트 방향 요약

- 목적: 개인 브랜딩/자기소개, 디스코드 관리자 경험 정리, 경험 공유 블로그
- 타깃: 디스코드 서버 멤버, 타 서버 인사 담당자, 현실 지인
- 톤앤매너: 미니멀 모던 + 퍼스널 컬러 `#87CEEB`
- 구조: 멀티페이지 (정적 우선 출시)
- 언어: 한국어/일본어/영어 3개 국어

## 2) 사이트맵

```text

├─ /ko
```

## 3) 글로벌 레이아웃

- 고정 헤더: 로고, 메인 내비게이션, 언어 전환, CTA(연락)
- 본문: 페이지별 콘텐츠
- 푸터: 저작권, 소셜 링크, 빠른 링크
- 공통 UX:
  - 모든 페이지에서 언어 유지 이동
  - 모바일 우선 반응형 (브레이크포인트: 640/1024/1440)
  - 접근성: 색 대비, 키보드 포커스, `aria-label` 기본 적용

## 4) 페이지별 와이어프레임

### A. Home (`/[lang]/`)

1. Hero
   - 이름/가명, 한 줄 소개, 핵심 역할 태그
   - CTA: `경력 보기`, `블로그 보기`
2. Brand Snapshot
   - 강점 3개 카드 (운영, 커뮤니케이션, 자동화)
3. Featured Career
   - 대표 경력 3\~5개 요약
4. Latest Posts
   - 최신 글 3개 카드
5. Quick Contact
   - Discord/GitHub/기타 링크

### B. Career (`/[lang]/career`)

1. Page Intro
   - 경력 페이지 설명 + 총 경력 요약 수치
2. Filter Bar
   - 카테고리, 직책, 규모, 기간 필터
3. Timeline
   - 기간순 카드/노드
   - 카드 정보: 서버명, 포지션, 역할 요약, 기간
4. Highlight Panel
   - 승진/리모델링/핵심 사례 요약

### C. Blog List (`/[lang]/blog`)

1. Category Tabs
   - 디스코드 운영 / 개발 기록 / 개인 회고 / 공지·업데이트
2. Search + Tag
   - 제목/요약 검색, 태그 선택
3. Post Grid
   - 썸네일(선택), 제목, 발행일, 카테고리, 요약
4. Pagination
   - 페이지네이션 또는 무한 스크롤(정적 우선은 페이지네이션 권장)

### D. Blog Detail (`/[lang]/blog/[slug]`)

1. Article Header
   - 제목, 발행일, 카테고리, 태그
2. 본문
   - 마크다운 렌더링
3. Related Posts
   - 같은 카테고리 2\~3개
4. Prev/Next
   - 이전/다음 글 링크

### E. Contact (`/[lang]/contact`)

1. Contact Methods
   - Discord, GitHub, 기타 채널
2. 안내 문구
   - 응답 가능 시간/용도
3. 향후 확장 슬롯
   - 동적 문의 폼 자리(2차)

## 5) 내비게이션 정책

- 1차 메뉴: Home / Career / Blog / Contact
- 보조 메뉴: 언어 전환, 외부 링크
- 현재 페이지 강조 + 스크롤 위치 기억(모바일 메뉴 포함)

## 6) 콘텐츠 정책 (초안)

- `personal_info.csv`: 자기소개/프로필 섹션으로 분해 매핑
- `career_data.csv`: 타임라인 데이터 원본
- 원문 유지 원칙을 따르되, 페이지 가독성을 위해 문단 분할/줄바꿈만 편집

## 7) 1차 출시 완료 기준

- 3개 언어 라우팅 동작
- 4개 주요 페이지 렌더링
- CSV 기반 Career 데이터 표시
- 블로그 카테고리 목록 + 글 상세 이동
