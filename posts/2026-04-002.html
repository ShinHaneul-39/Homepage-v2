Career 필터 성능 개선 기록

필터 조합이 늘어났을 때 UI 반응성을 유지하기 위해 적용한 최적화 포인트를 기록했습니다.

문제

필터 항목이 늘어나면서 조합 변경 시 렌더링이 눈에 띄게 느려졌습니다.

개선 방식

  • 원본 데이터는 불변으로 유지
  • 필터 상태 변경 시 필요한 계산만 재실행
  • 정렬 기준을 명시해 동일 조건에서 결과 순서 고정

회고

코드 복잡도를 크게 올리지 않고도 체감 반응성을 개선할 수 있었습니다.

Problem

As filter options increased, each state change caused visible UI delays.

Approach

  • Kept source data immutable
  • Recomputed only the required slices on state change
  • Enforced deterministic sort order for stable results

Retrospective

The UI became noticeably faster without introducing heavy architectural changes.
